Changelog¶

All notable changes to this project will be documented in this file.

The format is based on Keep a Changelog and this project adheres to Semantic Versioning.

1.1.0 - 2021-06-08¶

Compare with 1.0.91

Added¶

  • Addig logger to files that didn’t have it (1f5efeb by Lachlan Grose).

  • Adding background page for documentation (4d71586 by Lachlan Grose).

  • Adding option to perturb geometries from m2l (9c5f3fb by Lachlan Grose).

  • Adding scalar field class for visualising regular grid (98888e6 by Lachlan Grose).

  • Adding thickness probabilities to m2l function (fb8a9a2 by Lachlan Grose).

  • Adding autocalculated min and max to lambda feature (cf271c4 by Lachlan Grose).

  • Adding analytical geological feature (c3c006f by Lachlan Grose).

  • Adding debugging documentation page (5f0726b by Lachlan Grose).

  • Adding argument to skip features for debugging m2l (295cbf3 by Lachlan Grose).

  • Adding ability to colour faults by displacement magnitude (3188ba5 by Lachlan Grose).

  • Adding displacement magnitude to fault print string (d046abb by Lachlan Grose).

  • Adding function to install equality constraints to builder (6538614 by Lachlan Grose).

  • Added splay/abut as constraint that is evaluated (7889671 by Lachlan Grose).

  • Adding series/fault attribute for geological model (c8d7094 by Lachlan Grose).

  • Adding nsteps attribute to model vis (9e51b98 by Lachlan Grose).

  • Adding automatic estimate of the solver tolerance using the bounding box geometry and 1e-10 as a reference (21bb923 by Lachlan Grose).

  • Adding attribute to model which contains list of faults (680100b by Lachlan Grose).

  • Adding fault topology calculation per location in model (90ed747 by Lachlan Grose).

  • Adding graphviz to development dockerfile (d119497 by Lachlan Grose).

  • Adding placeholder to rotate interpolation support (3d65e37 by Lachlan Grose).

  • Adding callback_function to add_isosurface this is called with verts, tri, name for each surface and can be used for exporting the surface in custom format or extracting vertices of a surface (0c0277f by Lachlan Grose).

  • Adding automatic splay fault/abut fault to map2loop model. (2e2a4df by Lachlan Grose).

Fixed¶

  • Fixing bug for flake8 (15afedc by Lachlan Grose).

  • Fixed bug in fault topology calculator (a95ad85 by Lachlan Grose).

  • Fixing topology calculator (f0038d1 by Lachlan Grose).

Changed¶

  • Changed tetra ordering as per roys code (c221c5a by Lachlan Grose).

  • Changed model viewer isosurfacing to only compute scalarfield once. has changed api for vis a lot (a247da4 by Lachlan Grose).

  • Changed regions from a property on the mesh to a lambda function of a position or array of positions (1d70618 by Lachlan Grose).

  • Changed some imports and debugging (ea43f28 by Lachlan Grose).

  • Changed plot to add in visualisation. (a07bdef by Lachlan Grose).

  • Changed imports to relative (826b83f by Lachlan Grose).

  • Changed how data are assigned to structural frame, gave structural frame [] operator to access feature (ece58f5 by Lachlan Grose).

  • Changed constant gradient to use normalised face normal. this should fix up problems with cg values being too high (5c86bc5 by Lachlan Grose).

  • Changed long to long long in cython definitions inline with int64 (eb04cf3 by Roy Thomson).

  • Changed how pli weights are implemented - regularisation is weighted by the area of the shared face. points and gradient points are weighted by the volume of the tetrahedron. (8bdff99 by Lachlan Grose).

  • Changed boolean for faultedfeature gradient evaluation (64730f3 by Lachlan Grose).

  • Changed storing of data for structural frame builder to use a list of lists rather than a dict, this can be passed to the fault segment more easily. in the future could consider removing the ‘gx’,’gy’ tags and just use frame index. (06c2900 by Lachlan Grose).

  • Changed interpolation weights to be a dictionary stored by the interpolation object that can be updated when calling setup interpolator. this means the interpolator can be rerun using the same weights (8513693 by Lachlan Grose).

  • Changed structural frame so that the gz coordinte can be analytical (5dda636 by Lachlan Grose).

  • Changed name of tests folder to examples (8b47c03 by Lachlan Grose).

  • Changed docker file to use same base as lavavu and fme init now requires importing submodules manually (c7c1cd2 by Lachlan Grose).
